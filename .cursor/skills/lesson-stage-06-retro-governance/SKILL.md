# Lesson Stage 06 - Retrospective and Governance

## Goal
Keep quality stable over long runs by turning repeated failures into rule/skill/script improvements.

## Cadence
- Per lesson: always write a short retro entry.
- Per cycle: every 10-15 lessons run a governance review.

## Per-lesson retro fields
- lesson id/slug
- error types encountered
- root causes
- fixes applied
- automation gaps
- recommended rule/skill updates

## Cycle review outputs
- top recurring error categories
- drift indicators (correctness/UI/release)
- proposed rule/skill/script changes
- approved actions with owners and deadlines

## Non-negotiables
- No repeated unresolved failure pattern across two cycles.
- Governance actions must map to concrete file updates.
